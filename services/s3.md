# S3 Storage

Implemented in `server/src/services/s3.service.ts`.

## Buckets and Folders

Objects are stored under the following logical folders:

- `templates/` ‚Äî legacy Template videos
- `characters/` ‚Äî legacy Character images
- `compositions/` ‚Äî legacy Composition output videos, and `Reel` scene stills (image_kenburns niches)
- `reels/` ‚Äî Reel output videos, thumbnails, and voice-variant re-renders
- `audio/` ‚Äî Reel scene narration + legacy Composition dialogue-line speech
- `subtitles/` ‚Äî Reel and Composition subtitle files
- `gameplay/` ‚Äî the gameplay clip pool (persistent library, not tied to any single reel/job doc ‚Äî see `gameplay-ingest.service.ts`)
- `voice-samples/` ‚Äî cached TTS voice-preview clips, one per (model, voice), generated once and reused forever (see `voice-sample.service.ts`)

`reels/`, `compositions/`, `audio/`, and `subtitles/` are the four folders
the [S3 reconciliation sweep](../operations/scheduling.md#3-s3-reconciliation)
covers ‚Äî everything in them should be referenced by a current `Reel` or
`Composition` document. `gameplay/` and `voice-samples/` are deliberately
excluded from that sweep since they're persistent caches/libraries, not
per-job output.

## Upload Helpers

- `uploadToS3(buffer, folder, filename, contentType)` ‚Äî the primitive; every other upload helper wraps this.
- `uploadVideo(buffer, folder, originalFilename)`
- `uploadImage(buffer, folder, originalFilename)`
- `uploadAudio(buffer, originalFilename)` ‚Äî always `audio/`
- `uploadSubtitles(content, compositionId)` ‚Äî always `subtitles/`, filename = `${compositionId}.ass|.srt` (the one upload helper that keeps a meaningful, ID-based filename instead of a random one)

`uploadVideo`/`uploadImage`/`uploadAudio` all generate a **random** filename
via `generateFilename` (timestamp + random suffix) ‚Äî the caller's
`originalFilename` is only used to infer the file extension. This matters
for cleanup: you can't infer which reel a `reels/`/`compositions/`/`audio/`
object belongs to from its key alone; only `subtitles/` keys are
ID-addressable. It's also why the [reconciliation
sweep](../operations/scheduling.md#3-s3-reconciliation) diffs against every
document's referenced URLs rather than trying to parse ownership out of keys.

## Lookup / Existence

- `listKeys(prefix)` ‚Äî every key under a prefix.
- `listKeysWithMeta(prefix)` ‚Äî same, plus `lastModified` (used by the reconciliation sweep's age-safety check).
- `keyExists(key)` ‚Äî HEAD check, no download (used by the voice-sample cache to avoid regenerating).
- `getS3KeyFromUrl(url)` ‚Äî extracts the key from a direct S3 URL *or* a CloudFront URL.

## Deletion

- `deleteKey(key)` ‚Äî delete by raw key.
- `deleteFromS3(url)` ‚Äî delete by full URL (direct S3 or CloudFront); resolves the key via `getS3KeyFromUrl` then calls `deleteKey`.

Neither of these is "the" delete path for a reel ‚Äî see
[docs/api/endpoints.md](../api/endpoints.md#maintenance--cleanup) for the
actual cascading delete (`deleteReel`) and the reconciliation sweep that
catches what the cascade can't.

## CDN

`cdnUrlFor(key)` returns the CloudFront URL if `config.cdnUrl` is set
(`CLOUDFRONT_URL` env), else falls back to the direct S3 URL. Used for
gameplay clips and voice samples; reel/composition outputs are uploaded with
direct S3 URLs (see `uploadToS3`'s return value).

## Presigned URLs

`getPresignedUrl` returns a temporary download URL for a stored object.
